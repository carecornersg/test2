    User Creation

    This guide will walk you through creating and managing user accounts.

    Overview

    User management is available to System Administrators only. This section covers how to create, edit, and manage user accounts for the CAS Web Portal.

    Role Permissions

    Different roles have different capabilities in CAS:

    Call Centre Staff

    • ‚úÖ View case dashboard (real-time status)
    • ‚úÖ Create and edit cases
    • ‚úÖ Start/Stop broadcast
    • ‚úÖ Manual AA assignment
    • ‚úÖ Cancel cases
    • ‚úÖ View AA profiles (read-only)
    • ‚ùå Cannot create completed cases (Data Entry)
    • ‚ùå Cannot access advanced reports
    • ‚ùå Cannot configure system settings

    AAYS Manager

    • ‚úÖ All Call Centre Staff functions
    • ‚úÖ Full reporting access (all dashboards)
    • ‚úÖ AA management (override availability, view qualifications)
    • ‚úÖ Data export (CSV)
    • ‚úÖ Configure assignment logic
    • ‚úÖ View case-related audit logs only
    • ‚ùå Cannot perform user management

    System Administrator

    • ‚úÖ All AAYS Manager functions
    • ‚úÖ User management (create/edit/deactivate users)
    • ‚úÖ System configuration (notifications, integrations)
    • ‚úÖ Full Audit Logs access

    Create User

    Step 1: Navigate to Users

    In the left-hand side menu, navigate to Users

    Step 2: Click Create User

    Click the Create User button

    Step 3: Fill In User Details

    Fill in the following information:

    • Name ‚Äì User's full name
    • Email ‚Äì Login email address
    • Role ‚Äì Select: AAYS Manager, System Administrator, or Call Centre Staff

    Step 4: Click Create

    Click Create to save the new user.

    The user will receive email with a link to create password.

    Edit User

    Step 1: Select User

    • Click on the user name in the Users list
    • Click Edit button

    Step 2: Update Details

    • Modify the user information as needed
    • Toggle off Active if you want to deactivate user

    What happens when a user is deactivated:

    • ‚ùå User cannot login to CAS
    • ‚ùå If user is an AA, they will not receive any broadcasts
    • ‚úÖ User records and history are preserved
    • ‚úÖ User can be reactivated anytime

    Step 3: Save Changes

    Click Update to apply changes

    ‚ö†Ô∏è Important: Users Cannot Be Deleted

    CAS does not allow permanent deletion of user accounts. Users can only be deactivated.

    Reasons:

    • üìã Audit Trail ‚Äì All user actions must be traceable for compliance
    • üîó Data Integrity ‚Äì Cases and logs reference user accounts

    If you need to remove a user, simply deactivate them. Their account will remain in the system but they will not be able to access CAS.

    Troubleshooting

    User can't login: Check if user status is Active, try resetting password

    Wrong permissions: Verify the correct role is assigned

    Not receiving emails: Check email address is correct, check spam folder
